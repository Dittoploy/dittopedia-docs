# Docker & Trivy

## Build Docker

Chaque service dispose d'un `Dockerfile` multi-stage optimisé :

```dockerfile
FROM node:22-alpine AS builder
WORKDIR /app
COPY package.json bun.lockb ./
RUN bun install --frozen-lockfile
COPY . .
RUN bun run build

FROM node:22-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["node", "dist/main.js"]
```

## Scan de sécurité Trivy

Avant le push vers ECR, chaque image est scannée par **Trivy** :

```groovy
stage('Trivy Scan') {
    steps {
        sh 'trivy image --exit-code 1 --severity HIGH,CRITICAL dittopedia-back'
    }
}
```

## Push vers ECR

```groovy
stage('Push ECR') {
    steps {
        sh '''
            aws ecr get-login-password | docker login --username AWS --password-stdin $ECR_REGISTRY
            docker tag dittopedia-back $ECR_REGISTRY/dittopedia-back:$BUILD_NUMBER
            docker push $ECR_REGISTRY/dittopedia-back:$BUILD_NUMBER
        '''
    }
}
```
